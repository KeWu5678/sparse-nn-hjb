# penaltypowers — vdp

Sweeps the **penalty power** of the PDAP atom `σ(z)^p` on the smooth Van der Pol
value function (`VDP_beta_0.1_grid_30x30`, a filled 30×30 grid over [-3,3]²). The
atom power sets the nonconvex coefficient-penalty exponent `q = 2/(p+1)`: higher
power ⇒ smaller `q` ⇒ more concave penalty ⇒ sparser selection. Smooth target — the
counterpart of the switching-set study in `../../02_pendulum/frac_exp_penalty`, where the same knob
reverses. Findings are in `results.md` (generated by `analysis.py`); the VDP
optimal-control problem is in `src/OpenLoop/vdp` (Azmi–Kalise–Kunisch).

## Sweep axes

| axis | values |
|---|---|
| `data` | vdp |
| `model.kind` | signed |
| `model.insertion` | finite\_step |
| `model.activation` | every homogeneous activation in `src.config.activations` (relu, leaky\_relu, abs\_act, cubic, x\_absx, quartic, relu2, leaky\_relu2\_\*\_sphere, smoothy\_relu\_sphere, …) |
| `model.power` | 2, 2.01, 3, 4, 5 |
| `model.gamma` | swept (log-penalty nonconvexity) |
| `model.loss_weights` | [1,0] (L2, value only), [1,1] (H1, value+gradient) |

Fixed: `data.normalize=true`, `env.seed=42`. VDP is smooth, so the eval is the
**global** default (relative L2/H1) — no region split. Each config writes a Run
Record and a `result_*.pkl` fit artifact.

Score metric: `rel H1 × neurons` (lower is better). The penalty exponent is
`q = 2/(power+1)`, so `power = {2, 3, 4, 5}` gives `q = {0.67, 0.50, 0.40, 0.33}`.
The three representatives carried through the Key Finding in `results.md` are **ReLU
at powers {2, 3, 5}** (low / mid / high).

Canonical command:

```bash
make penaltypowers DATA=vdp
```

Executable defaults live in `conf/experiment/penaltypowers.yaml`.
