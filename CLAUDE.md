## Project Overview
Sparse shallow-network framework for solving Hamilton-Jacobi-Bellman (HJB)
equations from value/gradient samples, using a primal-dual proximal algorithm
(PDAP) with non-convex regularization and a semismooth Newton (SSN) outer solve.

## Plotting Style
Publication figures follow one house style. `PALETTE` and
`apply_publication_style()` live in `src/plotstyle.py` — import from there
(experiment scripts alias it as `_apply_publication_style`); do not re-declare
them per script. Save helpers (`_finalize_figure`/`_save_png`) remain local to
each `analysis.py`. The `scientific-plotting` / `scientific-figure-making`
skills cover the same conventions.

- **Palette** (never use matplotlib defaults):
  `blue_main #0F4D92`, `teal #42949E`, `red_strong #B64342`,
  `neutral #CFCECE`, `violet #9A4D8E`.
- **Typography**: serif (`CMU Serif` → `Computer Modern Roman` → `cmr10` →
  `DejaVu Serif`), base `font.size` 12, `mathtext.fontset = cm`,
  `text.usetex = False` (math via mathtext so figures build without a TeX
  install).
- **Axes**: hide top and right spines, `axes.linewidth` 1.0,
  `legend.frameon = False`, `axes.formatter.use_mathtext = True`.
- **Frontier plots are the one exception** (neuron/H1 frontiers): boxed axes
  (all four spines), dotted grid, dash-dot lines with dark-edged markers, and a
  framed legend row centred **below** the axes — apply via
  `src.plotstyle.style_frontier_axes(ax)` and save with `bbox_inches="tight"`.
- **Export**: save **only** `.png` at `dpi=300`. Current consumers use PNG —
  the thesis (`papar/Mthesis.tex`, `\includegraphics{plot/*.png}`) and the
  markdown result docs — so do not generate companion PDFs unless explicitly
  requested. `svg.fonttype = none`. Line plots `tight_layout(pad≈2.0)`; 3D
  surfaces use `bbox_inches="tight"` instead.
- **Figure sizes** in use: 3-panel row `(15, 4.6)`, single 3D surface
  `≈(4.2, 4.0)`, scatter `(8.5, 6)`, 2-panel time series `(14, 5)`.
- **Subfigures**: when a result needs multiple panels, save each logical panel as
  its own `.png` and arrange the subfigures in markdown/LaTeX tables or
  subfigure blocks. Use a composite multi-panel PNG only when the panels must
  share axes/colorbars or the user explicitly asks for one.
- **No in-figure titles** — on *any* plot, not just surface panels. Axis names
  may be shown, but all identifying/quantitative info (which series/activation,
  neuron counts, percentages, per-panel labels) belongs in the markdown/LaTeX
  caption, which is the single information source — not in-figure titles or text
  annotations.
- **3D plots** (value surfaces, weight scatters): view `elev=15, azim=-105` so the
  vertical axis sits on the **left** (uniform across every 3D figure); **transparent
  panes** — no grey background walls — via `pane.set_facecolor((1,1,1,0))` and
  `pane.set_edgecolor((0,0,0,0))`, with a faint grid (`color 0.85, lw 0.5`);
  **sparse ticks** (≈3 per axis) and **no axis names** (the caption labels the
  panel); save with `bbox_inches="tight"`.
